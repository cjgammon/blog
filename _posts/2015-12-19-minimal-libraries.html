Recently I've been interested in libraries, particularly rendering libraries for drawing content on the screen, like Snap.SVG and Pixi.js. I've also been a big fan of Greensock's animation library, GSAP. And while they still have TweenLite they also used to have a Nano library which was even more optimized for things like banner ads when file size needs to be under a specific limit. While file size is obviously important, with internet speeds we have today I feel it's often not as prioritized as it used to be but there are still cases when it's the deciding factor on whether to use a library.

So I started looking at my rendering libraries and they're often filled with a lot of extras that I don't need. This made me wonder how light of a library I could get if it only focused on rendering and nothing else with a razor like focus on light file size. So I started making them:

Element Libraries

I've named them after elements suggesting their stripped down nature focusing on the most basic necessities.

Hydrogen

Hydrogen is a lightweight canvas rendering library and display list that is around 1k in file size.

Carbon

Carbon is a lightweight SVG drawing library also around 1k.

I just started playing around with this and haven't used them extensively yet, but I think it's a neat idea. Let me know what you think, are these valuable? What other kind of situation would you want a minimal library for?
